# Grass Art: TEST

이 프로젝트는 GitHub의 Contributions Graph(잔디)를 활용하여 2025년 3월~5월 영역에 **"TEST"** 글자를 그리는 자동화 프로젝트입니다.

## 🛠 작동 원리 (How it works)

GitHub 잔디는 커밋 메시지에 포함된 **작성일(Author Date)** 정보를 기준으로 채워집니다. 이 프로젝트는 Git의 환경 변수를 조작하여 실제 시간이 아닌 과거의 특정 시점으로 커밋을 생성합니다.

### 1. 과거 날짜 커밋 생성
커밋 시 아래와 같은 환경 변수를 주입하여 시간을 과거로 고정합니다.
- `GIT_AUTHOR_DATE`: 커밋이 작성된 시간 (잔디가 찍히는 날짜)
- `GIT_COMMITTER_DATE`: 커밋이 저장소에 기록된 시간

### 2. 잔디 농도(진하기) 조절
GitHub 잔디는 해당 날짜의 **커밋 횟수**에 따라 4단계의 초록색 농도로 표현됩니다.
- **연한 초록:** 커밋 1~3개
- **진한 초록:** 커밋 15개 이상 (가장 진한 단계)
- 이 프로젝트는 글자를 선명하게 부각하기 위해 각 날짜마다 **하루 15개 이상의 커밋**을 생성하도록 설정되었습니다.

### 3. 글자 맵핑 (Pattern Mapping)
GitHub 잔디는 7행(일요일~토요일) 구조로 이루어져 있습니다. 이를 0과 1로 이루어진 행렬(Matrix)로 설계하여 "TEST" 글자 모양이 나오도록 특정 날짜에만 커밋을 수행합니다.
